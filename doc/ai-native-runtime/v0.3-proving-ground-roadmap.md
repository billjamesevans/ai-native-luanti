# v0.3 AI-Native Proving Ground Roadmap

Status: next milestone after v0.2 product-loop closure.

## Direction

v0.2 closed the first public-safe product loop: clean `ai_runtime` profile
verification, operator status/task control, low-power evidence, compatibility
inventory discovery, staged apply smoke, and stricter Minecraft-parity target
bands.

v0.3 should turn that into a stronger proving ground. The goal is not a content
pack and not a private family-server fork. The goal is a repeatable side-by-side
AI-native server lane where agents can do more useful work under explicit
budgets, rollback, audit, operator controls, and benchmark gates.

Compatibility/import remains behind runtime gates. It can expand only when the
runtime proves the apply path is bounded, reversible, observable, and
public-safe.

## Public-Safe Boundary

The milestone must not commit or retain:

- private family worlds or player-private data;
- `spacebase`, `themepark`, `disneyland100`, or showcase builds;
- copied Minecraft assets, server jars, marketplace content, or closed gameplay
  data;
- provider prompts, API keys, host credentials, or private payloads;
- live-family-world mutation from compatibility/import tooling.

The family server remains the proving ground for ideas, not the source tree for
the main fork. Generic runtime primitives can move into the fork only when they
have public-safe examples, focused tests, benchmark coverage, and clear operator
controls.
